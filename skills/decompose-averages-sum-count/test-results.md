## Failure Analysis

No failures. All binary cases passed.

**Decoy strength assessment:**

- `should-not-trigger-01` (additive SUM): This is the most important decoy category. Many practitioners who understand the average-of-averages problem will still wonder "does this apply to my sum?" The answer is no, and the B section is clear. Risk of false trigger is low because the metric (revenue) is clearly described as a sum, not a ratio. If a model triggers on "rollup across regions" alone without checking whether the metric is a ratio, this would be a false trigger.
- `should-not-trigger-02` (COUNT DISTINCT): This is the most dangerous decoy. The symptom (aggregation doesn't work across tiers) is the same; only the root cause differs. COUNT DISTINCT is non-decomposable in a stronger sense than AVERAGE — there is no SUM+COUNT fix available. A model that triggers on "aggregation doesn't behave across tiers" without distinguishing AVERAGE from COUNT DISTINCT would produce a false trigger. The phrase "COUNT DISTINCT" is the key differentiator; the A2 section does not mention COUNT DISTINCT as a trigger signal.
